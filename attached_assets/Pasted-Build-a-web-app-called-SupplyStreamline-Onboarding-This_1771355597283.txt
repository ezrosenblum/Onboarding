Build a web app called SupplyStreamline Onboarding. This app is an internal calling + email + tracking system used to onboard vendors and buyers, but we will build Vendor pipeline first and then duplicate it for Buyer once Vendor is working. Keep the code structured so Buyer can be duplicated with minimal changes (same screens/logic, different prompts/templates).

Tech choices

Use Postgres database.

Use a modern stack you recommend for Replit (e.g., Next.js + API routes + Prisma/Drizzle). Keep it simple and reliable.

Role-based authentication with separate logins.

Roles (3)

Admin: sees everything, config, uploads, assigns batches, dashboards later.

Vendor Caller: only sees vendor pipeline workspace, their assigned leads, their notes/calls/emails.

Buyer Caller: will be added later by duplicating vendor pipeline.

For Stage 1, implement Admin + Vendor Caller fully. Add Buyer role scaffolding (role enum + route protection) but you do NOT need to build Buyer UI yet.

C) Upload XLSX (Admin)

Create an Admin page: Upload Leads

Upload an XLSX file (Google Maps scraped leads).

Show a preview of the first ~20 rows.

Provide a simple “field mapping” UI:

Admin selects which spreadsheet columns map to our standard fields (company_name, phone, scraped_email, website, full_address, city, state, zip, place_id, cid, gmb_url, rating, reviews_count, hours_raw, category_keyword, timezone).

Admin can ignore other columns.

Admin selects the pipeline type for the upload (Vendor or Buyer). For Stage 1, allow selecting but only Vendor will be used in caller views.

Import into Postgres.

Dedupe: if place_id already exists for the same pipeline_type, skip or update (choose a safe approach: skip duplicates and report counts).

After import, show a summary: imported count, skipped duplicates count.
