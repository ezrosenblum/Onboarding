B) New table: signup_events (audit log)  [RECOMMENDED, implement]
Columns:
- id (pk)

- eventType (text, not null)                // e.g., 'SIGNUP_CONFIRMED', 'MANUAL_MARK_SIGNED_UP'

========================================================
2) WEBHOOK ENDPOINT (NO AUTH SESSION; SECRET HEADER)
========================================================

Add endpoint:
POST /api/signup/webhook

Security:
- Use a shared secret from Replit Secrets: SIGNUP_WEBHOOK_SECRET
- Require header: X-SS-Webhook-Secret: <secret>
- If missing or incorrect => 401

Request payload:
Minimum:
{ "lead_token": "<uuid>" }

Recommended supported fields:
{
  "lead_token": "UUID-string",
  "event": "signup_confirmed",
  "email": "user@email.com",
  "member_id": "12345",
  "confirmed_at": "2026-02-18T15:22:10Z",
  "idempotency_key": "optional-unique-id"
}

Server behavior:
A) Validate payload
- if lead_token missing => 400
B) Find lead by lead_token
- If not found => return 404 (OK) OR 200 accepted to avoid probing; choose ONE and be consistent.
  Preferred: 404 for now (internal), but make response generic (no extra info).
C) Idempotency:
- If lead.status_signup already SIGNED_UP:
  - insert signup_events anyway? (optional) OR skip insert if duplicate idempotency_key seen
  - return 200 { ok:true, already_signed_up:true }
D) Atomic update in a DB transaction:
- Update lead:
  - status_signup = SIGNED_UP
  - signed_up_at = confirmed_at if provided else now()
  - signed_up_email = payload.email if present
  - signed_up_user_id = payload.member_id if present
  - signup_source = 'webhook'
- Insert signup_events row:
  - event_type = 'SIGNUP_CONFIRMED'
  - payload_raw = full request payload
  - source_ip = req.ip (or header-forwarded value if present)
  - user_agent = req.get('user-agent')
  - idempotency_key = payload.idempotency_key (if any)
Return:
- 200 { ok:true, lead_id, already_signed_up:false }

3) ADMIN MANUAL OVERRIDE (REQUIRED)
========================================================

Admin-only API:
POST /api/admin/leads/:id/mark-signed-up

Auth:
- Must be logged in AND role=Admin, server-side enforced.

Body:
{
  "email": "optional",
  "signed_up_at": "optional ISO timestamp"
}

Behavior (transaction):
- Set lead.status_signup = SIGNED_UP
- signed_up_at = provided or now()
- signed_up_email = provided or keep existing
- signup_source = 'manual'
- Insert signup_events:
  - event_type = 'MANUAL_MARK_SIGNED_UP'
  - payload_raw = { email, signed_up_at, admin_user_id }
Return 200 with updated lead signup fields.

Also add a read API for lead signup events (Admin-only, optional but helpful):
GET /api/admin/leads/:id/signup-events
Returns event list ordered newest first.
