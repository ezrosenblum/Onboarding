Add these fields to leads (if not already present):

email_last_sent_at (timestamp nullable)

Create a new table email_logs:

id (pk)

lead_id (fk)

user_id (fk) // who sent it

template_type (text) // SEND_INFO / FOLLOW_UP / UNREACHABLE_OUTREACH

to_email (text)

from_email (text default connect@supplystreamline.com
)

subject (text)

body_html (text)

sendgrid_message_id (text nullable)

status (text default SENT) // SENT/FAILED

created_at

id (pk)

created_at

3) Email Templates (V1)

Implement 3 templates for the VENDOR pipeline (Buyer later by duplication):

Template A: SEND_INFO

Rules:

Allowed only if:

Lead has at least 1 call_log

confirmed_email is present and non-empty

lead is NOT unreachable
Email:

To: confirmed_email

Subject: simple, professional

Body: “Pleasure speaking…” + short SS summary + CTA link with ref token

Template B: FOLLOW_UP

Rules:

Allowed only if:

Same rules as SEND_INFO (call log + confirmed_email)

At least one prior email_log exists for this lead with template_type SEND_INFO
Email:

To: confirmed_email

Subject/body: short follow-up + same ref link

Template C: UNREACHABLE_OUTREACH

Rules:

Allowed if:

lead.unreachable = true OR lead reached max retry attempts

AND email exists (prefer confirmed_email if present, else scraped_email)
Email:

To: confirmed_email ?? scraped_email

Subject/body: “We tried reaching you…” + “free right now” + ref link

All emails must send from:

connect@supplystreamline.com

5) Backend endpoints

Create endpoints:

POST /api/leads/:id/email/send with body { templateType }

Server validates gating rules

Builds email subject/body

Sends with SendGrid using SENDGRID_API_KEY

Stores email_log

Returns success + email_log item

POST /api/sendgrid/events

This will receive SendGrid Event Webhook payloads (array of events)

For each event:

Identify the lead by:

unique_args.lead_token OR a custom arg you set on send

OR by matching sg_message_id to email_logs.sendgrid_message_id

Store event in email_events

Update lead.status_email:

If open event: status_email = OPENED (unless already CLICKED)

If click event: status_email = CLICKED

IMPORTANT:

When sending email via SendGrid, include a custom arg / unique arg so events can map back:

custom_args: { lead_token: <token>, lead_id: <id> } (or SendGrid equivalent)

Do not put the lead ID in the email subject.
